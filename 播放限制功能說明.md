# 播放限制和可見範圍功能

本文檔說明新增至 FUXA 的播放限制和可見範圍功能。

## 功能概述

播放限制功能允許管理員控制用戶進入 FUXA 網站時可以訪問哪些視圖。適用於：
- 限制特定視圖僅供特定用戶訪問
- 基於用戶角色控制訪問權限
- 為不同用戶群組創建自定義儀表板

## 資料庫結構

### 1. 常數參數表 (constParameters)

`constParameters` 表儲存具有可見範圍控制的常數參數。

```sql
CREATE TABLE constParameters (
  name TEXT PRIMARY KEY,
  value TEXT,
  visibility_scope TEXT,  -- '全域', '角色', '使用者', '僅限本人'
  creator TEXT,
  created_at INTEGER,
  updated_at INTEGER
);
```

**可見範圍類型：**
- `global` (全域): 所有用戶可見
- `role` (角色): 僅特定角色可見
- `user` (使用者): 僅特定用戶可見
- `owner` (僅限本人): 僅創建者可見

### 2. 播放限制表 (playRestrictions)

`playRestrictions` 表控制哪些用戶/角色可以訪問特定視圖。

```sql
CREATE TABLE playRestrictions (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  type TEXT,              -- 'user' 或 'role'
  view_id TEXT,           -- 要限制的視圖 ID
  user_id TEXT,           -- 用戶 ID (當 type='user')
  role_id TEXT,           -- 角色 ID (當 type='role')
  creator TEXT,           -- 創建此限制的人
  created_at INTEGER,     -- 創建時間戳
  updated_at INTEGER      -- 最後更新時間戳
);
```

### 3. 視圖表更新 (views)

`views` 表已更新以包含可見範圍：

```sql
CREATE TABLE views (
  name TEXT PRIMARY KEY,
  value TEXT,
  visibility_scope TEXT   -- 新增: 控制視圖可見性
);
```

**可見範圍類型：**
- `global` (全域，預設): 所有用戶可見
- `role` (角色): 僅特定角色可見（配合 playRestrictions 表）
- `user` (使用者): 僅特定用戶可見（配合 playRestrictions 表）
- `owner` (僅限本人): 僅創建者可見

### 播放限制 API

所有端點都需要管理員身份驗證。

#### 1. 獲取播放限制

```http
GET /api/playrestrictions
GET /api/playrestrictions?viewId=v_12345
```

**請求頭:**
- `x-access-token`: JWT 令牌 (必需)

**查詢參數:**
- `viewId` (可選): 按視圖 ID 過濾限制

**響應:**
```json
[
  {
    "id": 1,
    "type": "user",
    "view_id": "v_12345",
    "user_id": "john",
    "role_id": null,
    "creator": "admin",
    "created_at": 1234567890000,
    "updated_at": 1234567890000
  }
]
```

#### 2. 創建播放限制

```http
POST /api/playrestrictions
```

**請求頭:**
- `x-access-token`: JWT 令牌 (必需)
- `Content-Type`: application/json

**請求體:**
```json
{
  "type": "user",
  "view_id": "v_12345",
  "user_id": "john",
  "role_id": null
}
```

**響應:**
```json
{
  "id": 1
}
```

#### 3. 刪除播放限制

```http
DELETE /api/playrestrictions/:id
```

**請求頭:**
- `x-access-token`: JWT 令牌 (必需)

**響應:**
```json
{
  "success": true
}
```

#### 4. 獲取當前用戶允許的視圖

```http
GET /api/playrestrictions/allowed-views
```

**請求頭:**
- `x-access-token`: JWT 令牌 (必需)

**響應:**
```json
{
  "allowed": true,
  "views": ["v_12345", "v_67890", "v_11111"]
}
```

## 配置

### 環境變數

在 server 目錄創建 `.env` 文件：

```bash
# 啟用播放限制檢查
PLAY_RESTRICTION_ENABLED=true
```

或在 `server/settings.js` 中配置：

```javascript
module.exports = {
  // ... 其他設定

  playRestrictionEnabled: true,
}
```

**預設值:** `false` (不檢查限制)

## 工作原理

### 1. 當播放限制被禁用時

如果 `playRestrictionEnabled` 為 `false` 或未設置：
- 所有用戶可以訪問所有視圖
- 不檢查任何限制
- 系統行為與之前相同

### 2. 當播放限制被啟用時

如果 `playRestrictionEnabled` 為 `true`：

1. 當用戶登錄並請求視圖時，系統檢查 `playRestrictions` 表
2. 視圖根據以下規則過濾：
   - **基於用戶的限制**: 如果存在 `type='user'` 且匹配 `user_id` 的限制
   - **基於角色的限制**: 如果存在 `type='role'` 且用戶具有該角色
3. 沒有任何限制的視圖對所有用戶可訪問
4. 前端只接收用戶允許訪問的視圖

### 3. 限制邏輯

```
視圖可訪問的條件：
1. 如果該視圖不存在任何限制 → 所有用戶可訪問
2. 如果該視圖存在限制：
   - 用戶有特定用戶限制 (type='user') → 可訪問
   - 用戶的角色匹配角色限制 (type='role') → 可訪問
   - 否則 → 不可訪問
```

## 使用範例

### 範例 1: 設置視圖的可見範圍

**設置視圖為全域可見（預設）：**
```javascript
// POST /api/projectData
{
  "cmd": "set-view",
  "data": {
    "id": "v_dashboard",
    "name": "Main Dashboard",
    "visibility_scope": "global",  // 所有用戶可見
    // ... 其他視圖屬性
  }
}
```

**設置視圖僅限特定角色：**
```javascript
// POST /api/projectData
{
  "cmd": "set-view",
  "data": {
    "id": "v_operators_panel",
    "name": "Operators Panel",
    "visibility_scope": "role",  // 需配合 playRestrictions
    // ... 其他視圖屬性
  }
}

// 然後創建角色限制
// POST /api/playrestrictions
{
  "type": "role",
  "view_id": "v_operators_panel",
  "role_id": "operators"
}
```

**設置視圖僅限創建者：**
```javascript
// POST /api/projectData
{
  "cmd": "set-view",
  "data": {
    "id": "v_my_private_view",
    "name": "My Private View",
    "visibility_scope": "owner",  // 僅限本人
    // ... 其他視圖屬性
  }
}
```

### 範例 2: 限制視圖給特定用戶

```javascript
// POST /api/playrestrictions
{
  "type": "user",
  "view_id": "v_dashboard_admin",
  "user_id": "admin_john",
  "role_id": null
}
```

結果: 只有用戶 "admin_john" 可以訪問 "v_dashboard_admin" 視圖。

### 範例 2: 限制視圖給特定角色

```javascript
// POST /api/playrestrictions
{
  "type": "role",
  "view_id": "v_operators_panel",
  "user_id": null,
  "role_id": "operators"
}
```

結果: 所有具有 "operators" 角色的用戶可以訪問 "v_operators_panel"。

### 範例 3: 同一視圖的多個限制

```javascript
// 限制 1
{
  "type": "user",
  "view_id": "v_critical_systems",
  "user_id": "admin_john",
  "role_id": null
}

// 限制 2
{
  "type": "role",
  "view_id": "v_critical_systems",
  "user_id": null,
  "role_id": "senior_operators"
}
```

結果: "v_critical_systems" 視圖可被以下人員訪問：
- 用戶 "admin_john"
- 任何具有 "senior_operators" 角色的用戶

## 安全考慮

1. **僅限管理員**: 所有播放限制管理端點都需要管理員身份驗證
2. **需要令牌**: 所有端點都需要有效的 JWT 令牌
3. **預設開放**: 預設情況下，播放限制被禁用以保持向後兼容性
4. **顯式啟用**: 必須顯式設置 `PLAY_RESTRICTION_ENABLED=true` 才能啟用

## 遷移注意事項

- **向後兼容**: 現有的 FUXA 安裝將繼續正常工作，無需任何更改
- **無數據丟失**: 新表在首次運行時自動創建
- **現有視圖**: 所有現有視圖預設對所有用戶保持可訪問
- **可選功能**: 必須顯式啟用 `playRestrictionEnabled` 才能使用限制
